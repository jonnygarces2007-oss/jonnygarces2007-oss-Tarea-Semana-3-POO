## Comparación entre Programación Tradicional y Programación Orientada a Objetos en Python

**Nombre del estudiante:** Jonny Javier Garcés Almeida

### Descripción del proyecto

Este proyecto tiene como objetivo comparar dos enfoques de programación utilizados en Python para resolver un mismo problema: el registro y visualización de información de mascotas.

La primera solución fue desarrollada mediante Programación Tradicional, utilizando variables, funciones y entrada de datos por teclado. En este enfoque no se emplean clases ni objetos.

La segunda solución fue desarrollada mediante Programación Orientada a Objetos (POO), utilizando una clase llamada `Mascota`, la cual contiene atributos y métodos que permiten organizar de mejor manera la información y las acciones relacionadas con cada mascota.

### Archivos incluidos

**Programación Tradicional**

* tradicional.py

**Programación Orientada a Objetos**

* mascota.py
* main.py

### Reflexión

Durante el desarrollo de esta actividad fue posible observar diferencias importantes entre la Programación Tradicional y la Programación Orientada a Objetos. En la Programación Tradicional el código se organiza mediante funciones y variables independientes, lo que resulta adecuado para programas pequeños y sencillos.

Por otra parte, la Programación Orientada a Objetos permite agrupar los datos y comportamientos dentro de una misma estructura llamada clase. Esto facilita la organización del código, mejora la reutilización y simplifica el mantenimiento cuando los programas crecen en tamaño y complejidad.

En conclusión, ambos enfoques permiten resolver el mismo problema, pero la Programación Orientada a Objetos ofrece una estructura más ordenada, flexible y escalable para el desarrollo de aplicaciones más grandes.
